# Об докладе

### Тема

**«LLM во фронтенде: от чата к Agentic UI на примере CopilotKit + AG-UI»**

### Аудитория

Frontend-разработчики (React/Next.js), знакомые с TypeScript, но не обязательно с AI.

### Цель

Показать, что подключение LLM к фронтенду — это не «прикрутить ChatGPT-виджет», а полноценный новый слой UI: агент
управляет интерфейсом, вызывает функции и рендерит компоненты. К концу доклада слушатель понимает, что такое **AG-UI**,
и может за вечер собрать аналог демо из репозитория.

### Ключевые сообщения (takeaways)

1. LLM на фронте — это **протокол** (AG-UI), а не конкретная библиотека.
2. **Tool calling** превращает чат в полноценный контроллер UI.
3. **Generative UI** = LLM выбирает не только текст, но и React-компонент с пропсами.
4. CopilotKit — самый быстрый способ начать; альтернативы: LangChain.js, MCP, Tanstack AI, A2UI.

# Структура доклада

### 1. Intro (2 мин)

- Боль: LLM во фронте часто = «плавающий виджет, который ничего не умеет».
- Тезис: настоящая ценность — когда LLM **управляет UI и вызывает функции приложения**.
- Анонс: Гранд-демо всего проекта прямо сейчас.

### 2. Показ демо-проекта: возможности CopilotKit (3 мин)

- **Цель**: Продемонстрировать возможности интеграции LLM во фронтенд через визуальный восторг.
- **Действие**: Череда быстрых промптов, показывающих ключевые фичи:
    1. **Смена title через чат**: AI меняет заголовок страницы (`changeHeading`).
    2. **Смена темы**: Мгновенное обновление CSS-фона (`changeBackground`).
    3. **Обновление данных**: AI пересчитывает и обновляет карточки статистики (`updateStats`).
    4. **Уведомления**: Отправка тостов (toast) прямо из чата (`sendNotification`).
    5. **Работа со списком задач**: Двусторонняя синхронизация (Shared State) между UI и AI (`agent.state.todos`).
    6. **Human-in-the-loop**: Агент запрашивает подтверждение/ввод пользователя (`useHumanInTheLoop`).
    7. **Контекст**: Демонстрация того, как агент знает о пользователе (`useAgentContext`).
- **Результат**: Зритель видит, что чат — это не просто виджет, а полноценный слой управления приложением.

### 3. Что такое AG-UI (Agentic UI) (2 мин)

- **AG-UI** — открытый протокол взаимодействия между **агентом (LLM)** и **UI** (аналог LSP для AI-агентов).
- Стандартизирует поток событий: `messages`, `tool calls`, `state updates`, `lifecycle`.
- Транспорт-агностичен (SSE/WebSocket), кросс-фреймворковый.
- В нашем репо: `@ag-ui/client@0.0.53` + `logMiddleware` в `route.ts` — наглядный «AG-UI на проводе».

### 4. Как подключить LLM к фронтенд-приложению + Разбор Demo #1 (3 мин)

Архитектура на примере репо:

```
React UI (CopilotChat) ──► /api/copilotkit (Hono) ──► CopilotRuntime ──► LanguageModel (OpenRouter/Claude)
         ▲ AG-UI events                          tools[]                ai-sdk middleware
```

Пошагово (код из `src/app/api/copilotkit/[...copilotkit]/route.ts`):

1. Любой LLM-провайдер через **Vercel AI SDK**: `createOpenAI({ baseURL: 'https://openrouter.ai/api/v1' })` — один и тот
   же код для OpenAI, Anthropic, локальной модели.
2. `wrapLanguageModel` + middleware (логирование токенов) — показать `tokenUsageMiddleware`.
3. `BuiltInAgent({ model, prompt, tools })` — агент.
4. `CopilotRuntime` + `createCopilotHonoHandler` — серверный эндпоинт.
5. На клиенте — `<CopilotChat />` из `@copilotkit/react-core/v2`.

**Вывод для зала:** подключение любой LLM = поменять `openrouter.chat('anthropic/claude-sonnet-4.6')` на нужную модель.

**Разбор Demo #1 (Weather):** Как промпт превратился в запрос к серверу и вернулся красивой карточкой.

### 5. Вызов инструментов на фронте через LLM + Разбор Demo #2 (3 мин)

Два типа tool-ов:

- **Backend tools** — `defineTool` + `execute` на сервере. Пример: `getWeatherBackend` в `route.ts`.
- **Frontend tools** — `useFrontendTool` в React, выполняются в браузере, имеют доступ к DOM/состоянию/колбэкам.

Из `src/shared/ui/copilot-chat.tsx` показать реальные tool-ы:

- `changeBackground` — LLM меняет CSS страницы.
- `sendNotification` — пушит тост в правый угол.
- `updateStats` — переписывает карточки дашборда.
- `celebrate` — запускает конфетти.

Ключ: **Zod-схема параметров** = типобезопасный контракт между LLM и кодом.

**Разбор Demo #2 (Theme):** Как LLM получила доступ к колбэкам React-компонента через `useFrontendTool`.

### 6. Формирование UI через LLM (Generative UI) + Разбор Demo #3 (3 мин)

- Идея: LLM возвращает не текст, а **структурированный вызов**, который рендерится как React-компонент.
- API: `useRenderTool({ name, parameters, render })` — рендер по статусам `inProgress | executing | complete`.
- Демо из репо: `WeatherCard` и `RecipeCard` рендерятся прямо в чате (`copilot-chat.tsx`, строки 21–111, 140–224).
- Wildcard-рендерер `name: '*'` — fallback для любых неизвестных tool-ов.
- `useConfigureSuggestions` — LLM сама подсказывает, что у неё можно попросить.
- Кратко: кастомизация чата (`CustomChatInput`, `CustomChatMessageView`, `WelcomeScreen`) — CopilotChat headless.

**Разбор Demo #3 (Recipe):** Магия `useRenderTool` — превращение JSON-ответа модели в интерактивный компонент.

### 7. Демо реального проекта: Управление документацией (Confluence-like) (2 мин)

- **Цель**: Показать, что эти же концепции (AG-UI, tools, GenUI) работают в большом production-проекте.
- **Действие**: Демонстрация системы управления документацией.
- **Фичи**:
    - Поиск по документам через чат.
    - Автоматическое создание/редактирование страниц документации.
    - Ассистент, который помогает структурировать контент.
- **Takeaway**: Это не просто игрушка, а мощный инструмент для корпоративного ПО.

### 8. Какие ещё инструменты есть (2 мин)

Краткий обзор экосистемы:

- **LangChain.js / LangGraph.js** — оркестрация сложных агентов на сервере.
- **MCP сервера** — подключаются через Middleware
- **Tanstack AI** — инструменты для работы с LLM на клиенте (совместим с Copilotkit).
- **A2UI** — Google's declarative, LLM-friendly Generative UI specification (совместим с Copilotkit).
